# Security Policy

このドキュメントは SkyWebPro のセキュリティ運用方針を定義します。

## サポート方針

- 最新の main ブランチを優先サポート対象とします。
- 脆弱性修正は可能な限り小さな差分で適用します。

## 報告方法

- 公開Issueに機密情報を書かないでください。
- 重大な脆弱性は管理者へDMまたは非公開連絡で報告してください。
- 再現手順、影響範囲、想定される攻撃シナリオを記載してください。

## 取り扱い方針

- 受領後に再現確認を行います。
- 対応方針を決め、必要なら公開前に修正版を準備します。
- 修正後は CHANGELOG に記録します。

## 実装上の基本方針

- 外部入力は常にエスケープまたはサニタイズする。
- URLは http/https のみ許可する。
- セッション情報はローカル保存のリスクを前提に扱う。
- ログやエラーメッセージに機密情報を含めない。

## 既知リスク

- ローカルストレージ利用に伴う共有端末リスク。
- 外部API仕様変更に伴う予期しない動作変化。
