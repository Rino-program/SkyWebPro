# Deploy Recovery Guide

GitHub Pages デプロイ失敗時の復旧手順です。

## 症状

- Actions の deploy ジョブが失敗する
- 公開URLで古い内容が表示される
- 404 が出る

## 確認手順

1. GitHub Actions で最新実行ログを確認する。
2. Pages の Source が GitHub Actions になっているか確認する。
3. 対象ブランチが main か確認する。
4. アセットパスが相対パスになっているか確認する。

## 復旧手順

1. ワークフロー失敗原因を修正する。
2. main に再度 push して再実行する。
3. Actions 成功後、数分待って公開URLを再確認する。
4. ブラウザキャッシュをクリアして再読み込みする。

## 典型原因

- ワークフローファイルの構文不備
- 参照ファイルの削除やパスミス
- 予期しない権限設定変更

## 予防

- PR で lint と test を先に通す。
- リリース前に README の公開手順を再確認する。
