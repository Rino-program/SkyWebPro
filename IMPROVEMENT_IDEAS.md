# SkyDeck 改善アイデア集（100+）

このドキュメントは、SkyDeck を「使いやすさ・信頼性・継続運用」の面で強化するための改善案を整理したものです。

## 1) UX / UI 改善

1. 初回ログイン時に 60 秒のツアー（主要タブ説明）を表示する。
2. 投稿入力欄に「下書き復元」バナーを出す（復元/破棄）。
3. 投稿成功時に「投稿へ移動」ボタン付きトーストを出す。
4. DM 画面で未読会話を上に固定表示する。
5. 設定画面に検索ボックスを追加し、設定項目を即時フィルタ可能にする。
6. 返信中は投稿欄背景色を変えてコンテキストを明確化する。
7. 通知カードに「既読にする」個別ボタンを追加する。
8. Discover/Following 切替時に軽いスケルトンを表示して体感速度を上げる。
9. モバイルで下部タブバーを提供し、サイドバー依存を減らす。
10. 空状態（Empty State）に次アクションを提案する CTA を追加する。

## 2) アクセシビリティ

11. すべてのアイコンボタンへ明確な aria-label を付与する。
12. モーダル表示時にフォーカストラップを実装する。
13. Esc キーで全モーダルを閉じられるよう統一する。
14. タブ切替に `aria-selected` / `role=tablist` を付与する。
15. 投稿カード内操作をキーボードだけで完結できる順序に最適化する。
16. 色だけで状態を示す箇所（いいね等）にテキスト/形状差を追加する。
17. 画像未読込時の代替テキストを改善する（投稿者名 + 文脈）。
18. 文字サイズ 200% でも崩れないレイアウト検証を追加する。
19. 「アニメーションを減らす」OS設定に従いアニメーションを抑制する。
20. スクリーンリーダー向けに通知件数のライブリージョンを最適化する。

## 3) パフォーマンス

21. 長いフィードを仮想スクロール化して DOM ノード数を抑える。
22. 投稿画像の lazy load に加えて decode="async" を付与する。
23. API レスポンスの正規化キャッシュ層を追加して再レンダリングを軽量化する。
24. タブ再訪時は差分更新のみ行う（完全再描画を避ける）。
25. CSS を機能単位に分割し、初期ロードを最小化する。
26. 画像サムネイルのサイズヒントを指定してレイアウトシフトを減らす。
27. 入力中検索は `requestIdleCallback` と debounce の併用で負荷平準化する。
28. 通知ポーリングを可視状態に応じて自動間引きする。
29. Service Worker で静的アセットをキャッシュし再訪を高速化する。
30. 主要操作（ログイン、ホーム表示）の計測を入れ p95 指標を可視化する。

## 4) セキュリティ / プライバシー

31. localStorage 保存セッションを Web Crypto で暗号化保存する。
32. 認証情報の保存有効期限（例: 7日）を設定する。
33. CSP の `unsafe-inline` を削減し nonce 方式へ移行する。
34. 外部リンク確認ダイアログにドメイン表示を追加する。
35. 共有端末向け「自動ログアウト（非操作30分）」を追加する。
36. ログイン時にパスワード貼り付け可否を設定化する。
37. エラーログ出力から機密値（JWT等）を自動マスクする。
38. DM 送信前に危険文字列（秘密情報の誤送信）簡易チェックを入れる。
39. セキュリティポリシー文書を README から分離し詳細化する。
40. 依存先ドメイン一覧と用途を設定画面に可視化する。

## 5) 信頼性 / エラーハンドリング

41. API 失敗時のエラー分類（認証/通信/制限）を統一する。
42. リトライ戦略（指数バックオフ + ジッター）を共通化する。
43. オフライン検知時は投稿操作を自動キューイングする。
44. 投稿失敗時に内容を確実に下書きへ退避する。
45. フィード取得中の二重リクエストを AbortController で中断する。
46. 401 発生時に「再ログイン導線 + 現在作業の保護」を明示する。
47. 無限スクロール末尾到達判定を Observer 化して漏れを減らす。
48. 画像アップロード失敗時に枚数単位で再試行できるようにする。
49. グローバル例外ハンドラでユーザー向けメッセージと内部詳細を分離する。
50. バージョン不一致時に移行ウィザードを表示する。

## 6) 機能拡張（投稿・タイムライン）
